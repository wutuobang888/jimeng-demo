# 即梦图片自动匹配排序工具（桌面版 MVP）

一个本地运行的 `Python + tkinter` 桌面工具，用于把即梦等平台下载后顺序混乱的图片，按你粘贴的提示词顺序自动匹配、复制、排序和重命名。

## 功能概览

- 直接粘贴提示词（每行一条）
- 支持添加多个图片来源
- 来源支持：
  - 文件夹（可递归扫描子目录）
  - ZIP 压缩包（可多选，自动解压到临时目录）
  - 文件夹 + ZIP 混合
- 自动从文件名提取语义文本并做模糊匹配
- 使用全局唯一分配（匈牙利算法）
- 输出排序后的新文件夹（不修改原图）
- 生成 `match_results.csv` 方便人工复核
- 低置信度自动标记

## 环境要求

- Python 3.10+

## 安装依赖

在项目目录执行：

```bash
pip install -r requirements.txt
```

## 运行

在项目目录执行：

```bash
python main.py
```

## 使用步骤

1. 打开程序
2. 在“提示词输入区”粘贴提示词文本（每行一条）
3. 添加图片来源：
   - 点击“添加文件夹来源”可逐个添加文件夹
   - 点击“添加 ZIP 来源（可多选）”可一次添加多个压缩包
4. （可选）选择输出目录
5. 点击“开始处理”
6. 程序完成后会弹窗提示输出目录

## 输出结果说明

程序会在输出目录创建一个新文件夹，命名规则如下（自动避免覆盖）：

```text
sorted_output/
sorted_output_1/
sorted_output_2/
```

输出内容包括：

- 按提示词顺序重命名后的图片，例如：
  - `001_提示词简写.png`
  - `002_提示词简写.jpg`
- `match_results.csv`

## CSV 字段说明

`match_results.csv` 至少包含以下字段：

- `目标序号`
- `目标提示词`
- `原文件名`
- `文件名提取文本`
- `匹配得分`
- `是否低置信度`
- `导出文件名`
- `来源路径`
- `来源类型`
- `原始相对路径`
- `状态`
- `错误信息`

## 压缩包处理说明

- 当前 MVP 支持 `.zip`
- 压缩包会自动解压到程序运行时临时目录
- 程序结束后临时目录自动清理
- 原始压缩包不会被修改

## 注意事项

- 原始图片不会被修改，仅复制到新目录
- 若提示词数量与图片数量不一致，程序会继续处理，并在日志和 CSV 标记未匹配项
- 若某些图片文件名提取文本为空，通常会被标记为低置信度
